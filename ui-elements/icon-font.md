### Icon Workflow ###

Periodically, the UX team will provide us with updated icons. Here is the workflow that needs to be followed to get these files in place

* **Updated font.zip sent from UX team** Currently, Bryce is creating these files
* **Regenerate the fonts** Unfortunately, the files that UX sends display correctly on a local webpage, but they do not display correctly when served via IIS. The only known fix is to regenerate the files.
  * Unzip the font.zip file
  * Point your browser to http://icomoon.io/app/#browse
  * If there are any icons loaded, click the reset icon in the lower righthand of the page
  * If nessasary, chose the trash can icon and delete any remaining icons
  * Click "Import Icons" button<
  * Navigate to the unzipped font directory, and choose the file called selection.json
  * Click the "Font ->" button
  * Under "Preferences" make sure:
    * Icon name is set to "concur-icons-{YYYYMMDD} (supply the current date)
    * Class prefix is "icon-
    * Generate Stylesheet variables for LESS
    * use Attribute Selectors
  * Under "Codes" make sure we start at "e000"
  * Click the "Download" button
  * Unzip the fonts downloaded from icomoon
  * SVN Replace the concur-icon font files in \static\fonts\ with the ones generated from icomoon
* **Update the icons.less file** Manually diff \static\less\cnqr-global\_components\_cnqr-icons.less against style.css provided with the fonts download from icomoon. Please be on the lookout for:
  * New icons
  * Altered char-codes
  * Renamed icon class names
* **Update the _variables-cnqr-icons.less file** Replace all char codes with char-code variables located in \static\less\cnqr-global\_components\variables-cnqr-icons.less so they can be refererence elsewhere if needed. Use the _variables.less provided from icomoon
* **Update IconFont.cshtml (this file)** Manually diff IconFont.cshtml with the demo.html file providede with the fonts downloaded from icomoon
* **Update the manual cachekilla for fonts** In static\less\cnqr-bootstrap-3.3.1\custom\_variables.less, update the @@font-icon-cnqr-cachekilla to todays date string
* run C:\Outtask\Tools\UnitTests\iconsaudit.wsf to determine if any icons changed names. Update script and code code as needed.
